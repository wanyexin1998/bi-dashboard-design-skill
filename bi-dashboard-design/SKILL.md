# BI Dashboard Design — Wyexin Visual DNA

把 Wyexin 从 16 张历史看板中沉淀的个人视觉语言,复用到每一张新看板上。产出两种形态:①直接可用的 HTML+ECharts 看板;②给 BI 工具搭建用的设计规范。两种形态遵循同一套 token 与规则。

## 风格一句话

**浅灰画布上的白卡片矩阵,居中字距标题,渐变柱形,数据标签常开,红线丈量目标差距,月蓝年橙,绿涨红跌,高密度但每卡只答一个问题。**

## 设计第一性原理(先想清楚再动手)

1. **看板是答题卡,不是图表相册**——每张卡回答一个明确业务问题,卡片标题就是问题的名词化。排版顺序 = 提问顺序:结论→进度→趋势→结构→责任→明细。
2. **企业经营看板的灵魂是目标管理四件套**:目标、达成、差距、时间进度。凡有目标的指标,必须能一眼看出"照这个节奏能不能完成"——这就是时间进度虚线存在的意义。
3. **颜色是语法不是装饰**:主题色定身份(哪条业务线),语义色定含义(涨跌/目标/差距),中性色定结构。同一含义永远同一颜色。
4. **视觉层级 = 数据层级**:越重要的数字越大、越深、越靠左上。装饰只出现在页头、KPI 卡底水印、丝带投影三处。
5. **密度靠秩序消化**:高密度是企业看板的宿命,用等高卡片、统一标题句式、分区丝带、镜像结构(月/年对称)让密度变得可扫读。
6. **数据标签常开换取轴的沉默**:标签直接标在图上,y 轴隐藏,网格线极淡——数字更近,噪音更少。
7. **自解释**:单位缀在标题尾「(万)」,口径写在脚注,可联动处放红字提示。

## 工作流

### Step 0 — 判断产出形态
- 用户要成品/演示/原型 → **HTML 模式**
- 用户要在 BI 工具里搭 → **规范模式**
- 不明确时问一句,或默认 HTML 模式(可兼作规范参考)。
- 内容清单缺失时(如只说"做个经营看板"),动手前先问三件事:**时间粒度**(日/周/月/实时)、**有无目标值**、**给谁看**——三问的答案决定范式与行型,视觉是内容的函数。

### Step 1 — 定主题(用户需求 > 业务域预设 > 默认蓝)
用户对主题/品牌色/背景有任何要求时,**以用户为准**,按 `design-tokens.md` §2.5 推导整套自定义 token(语义色与组件结构保持不变)。用户没提时按业务域取预设:

| 业务域 | 预设主题 |
|---|---|
| 集团经营、业绩、销售、通用 | `blue` 商务蓝(默认) |
| 医院、健康、商城、活动、监测 | `teal` 医养青绿 |
| 渠道/代理月报、沉稳汇报 | `navy` 藏蓝薰衣草 |
| 营销专项、大促、高注目临时看板 | `pop` 洋红青黄 |

### Step 2 — 排版(本 skill 的重心,读 `references/layouts.md`)
1. 选范式:驾驶舱 A / 日报 B / 监测专项 C / 分中心模板 D,画出章节骨架(每章一条丝带)。
2. **逐行套用行型库**(§2):KPI 一行放几个、趋势图占多宽、环图排名怎么组行——查表,不自由发挥。
3. **先数类目再定宽度**(§3 宽度算法):12 个月的组合图必须全宽,TOP15 排名必须够高,窄卡环图禁用引出标签。

### Step 3 — 逐卡设计
- 颜色、字体、间距:一切数值查 `references/design-tokens.md`(唯一事实来源)。
- KPI、进度环、目标条、表格、筛选器:查 `references/components.md`。
- 图表选型与 ECharts 配置:查 `references/charts.md`,系列颜色语义(灰=去年、珊瑚=目标、红线=达成率、月蓝年橙)不可违背。
- **造数规则(单一事实源)**:先把整板数据建成一个 `DATA` 对象,所有卡片取数同源;达成率、差额、占比、合计、同比一律公式派生、禁止手填;交付前勾稽——合计=分项之和、环图扇区和=对应 KPI、同一指标跨卡同值。

### Step 4 — 产出
- **HTML 模式**:以 `assets/starter-template.html` 为骨架起步——它已内置全部 CSS 变量、四主题切换、公共 ECharts 配置和签名组件样例。单文件交付,ECharts 走 CDN,数据可造但要业务上合理(达成率有高有低、有超前有滞后,才真实)。模板已内置几何 QA:交付时告诉用户「页面 URL 加 `?qa=1` 可自检」——自动检测横向溢出、卡片参差、图表文本越界、异常字号,标红并出报告;Agent 自己有浏览器能力时应先带 `?qa=1` 跑一遍再交付。
- **规范模式**:输出 Markdown 设计规范,结构为:主题与色值表 → 页面分区图(文字画出章节骨架) → 逐卡片规格(图表类型/系列颜色/标签开关/组件样式要点) → 全局规则清单。所有色值直接给十六进制,让用户能在 BI 工具里逐项配置。每个样式点标注落地性:`[原生]`(工具直接支持)/`[近似]`(给替代方案,如双率线悬浮→双轴+辅助线、标题字距→全角空格、渐变柱→纯色)/`[舍弃]`——不给用户开工具做不到的药方。

### Step 5 — 自检清单(交付前逐条过)

**视觉 DNA:**
- [ ] 画布 #F7F8FA(或用户指定背景),卡片圆角 8px 细描边,无厚投影
- [ ] 页面主标题字距拉开(letter-spacing ≥ .3em),章节用「— x x —」丝带
- [ ] 每卡标题居中 **16px**(强调可 18px),单位缀在标题尾
- [ ] 凡柱必渐变;数据标签常开 11px;y 轴隐藏或极淡
- [ ] 有目标的地方看得见:目标、差距、达成率、时间进度虚线
- [ ] 月蓝年橙;去年灰;目标珊瑚;达成率红线
- [ ] 涨跌:绿=向好红=向差(↑#18A058 / ↓#E64C45;流失率等逆向指标下降标绿),全板一致
- [ ] 一板一主题(用户指定优先),分类色按 tokens 色板顺序取
- [ ] 数字千分位、tabular-nums、右对齐(表格内)

**排版与溢出(严格执行 `layouts.md` §6):**
- [ ] 每行卡数符合行型库;同行卡片等高;一屏 ≤9 卡
- [ ] 类目数与图宽匹配(组合图每类目≥90px;TOP-N 每行≥28px;环图分类≤6)
- [ ] `containLabel:true`;柱顶标签留头(`yAxis.max≈1.15×`);横向条 `grid.right≥48`
- [ ] 长类目名截断+tooltip 全名;大数值换「万/亿」;KPI 数字不溢出卡片
- [ ] 每个图表容器固定高度,`resize` 监听已挂
- [ ] 有渲染能力则 1920/1440 两档截图目检;无渲染能力则 Node 执行全部 option 构建函数验证无异常无 NaN
- [ ] 数字勾稽:合计=分项和、环图扇区和=对应 KPI、同一指标跨卡同值
- [ ] 已提示用户(或已自行执行)`?qa=1` 几何自检

## 适用边界(超界先声明,不许硬套)

本 skill 面向**浅色、中文、桌面端**的企业经营看板。三种邻近场景按降级规则处理并提前告知用户效果差异:深色大屏→仅按 `design-tokens.md` §2.5 深底变体执行,组件结构与图表语言保留;英文看板→字距签名失效,主标题改 `letter-spacing:.06em`;移动端→按 `layouts.md` 通用规则单列化并砍明细表。完全超界的诉求(海报、长图文、印刷物)直说不适用。

## 反模式(见到就改)

彩虹配色的柱状图;3D/阴影图表;纯白页面底;每个数字配一个图标;饼图超过 6 类;红涨绿跌与绿涨红跌混用;y 轴+数据标签同时开;卡片高低参差;把"目标"画成和"当期"同色;用暗黑大屏风格做经营日报。

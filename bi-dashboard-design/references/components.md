# 组件规范 — Wyexin BI 组件词汇表

从 16 张历史看板中提炼的组件库。每个组件给出:何时用、解剖结构、关键样式。CSS 类名与 `assets/starter-template.html` 对应。

## 目录
1. 页头与分区(header / ribbon / banner)
2. KPI 家族(7 种变体)
3. 目标与进度家族(签名组件)
4. 表格
5. 筛选与交互控件
6. 文本注释组件
7. 装饰规则

---

## 1. 页头与分区

### 1.1 页头药丸(默认页头)
- 结构:居中药丸容器(白色渐变底、下缘柔和投影),内放字距拉开的粗体标题;LOGO 靠左;日期/全局筛选靠右。
- 样式:标题 20–22px/700/`letter-spacing:.35em`/`--text-1`;药丸底 `linear-gradient(180deg,#FFFFFF, #F2F5FA)`,圆角全圆,左右各留一段渐隐横线,可叠六边形纹理水印。
- 何时用:驾驶舱、日报等常规看板。

### 1.2 渐变横幅页头(强调版)
- 整幅横幅 `135°` 主色渐变(深→亮),白色字距标题居中,筛选器置横幅右端(白色半透明底)。
- 何时用:汇报型看板、专项分析首屏;积分/监测类可用深色金属质感渐变。

### 1.3 分区丝带(章节标题,签名元素)
- 居中药丸,文字「— 住 院 分 析 —」:16px/700/`letter-spacing:.5em`,两端「—」;丝带底白色带极淡投影,压在画布上。
- 一张长滚动看板 = 若干章,每章一条丝带。丝带是目录,命名用业务域名词(「业绩总览」「销售分析」「渠道分析」)。

### 1.4 分中心横幅(模板化复制场景)
- 整行蓝色渐变圆角横幅写分支机构名(如「北京中心」),其下跟一组固定面板;多个分支纵向复制同构模板。
- 何时用:多院区/多分公司同构监控。

## 2. KPI 家族

通用规则:标签在上(12px `--text-3`),数字在下(30–38px/700/`--text-1`/tabular-nums);涨跌幅 12px 带 ↑↓,绿涨红跌;单位缀在标签尾「(万)」。

| # | 变体 | 结构 | 何时用 |
|---|---|---|---|
| 2.1 | 裸数字卡 | 白卡内仅标签+大数字 | 密度型简报、次要指标 |
| 2.2 | 涨跌卡 | 标签+大数字+同比/环比行(↑↓彩色) | 默认 KPI |
| 2.3 | 日/月/年三联卡 | 一张卡三列,列首为维度小标题带 `--tint` 底色条;或三张卡并排各配色(蓝=日、青=月、橙=年) | 驾驶舱首屏 |
| 2.4 | 强度编码 chip 行 | 一行 6–9 个圆角 chip,底色从 `--primary` 深端到 `--tint` 递减,值在 chip 下方;色深 = 指标优先级 | 日报顶栏漏斗式指标链 |
| 2.5 | 浅底 wash 卡 | 卡底 `--tint`,可叠淡水印图标 | 需要成组区隔的 KPI 组 |
| 2.6 | 图标纵列卡 | 圆圈线性图标(主题色描边)+标签+大数字,纵向堆叠成列;日/月/年三列各用蓝/青/黄图标 | 流程监测、履约类 |
| 2.7 | KPI+迷你趋势同框 | 左半 KPI,右半迷你面积图/折线,共享一张卡 | 指标+走势一体表达 |

## 3. 目标与进度家族(本体系的灵魂)

企业经营看板的核心叙事是「目标—达成—差距—时间」。以下组件必须成套理解:

### 3.1 进度环 + 指标行(签名)
- 左:进度环(轨道 `#EEF1F6`,进度弧主题色渐变、圆头;中心写维度名「月业绩」)。
- 右:指标行列表,右对齐数值:`目标`(`--text-1`)、`差额`(`--down`)、`达成率`(`--down` 或 `--primary`)、`剩余天数`(`--text-1`)、`时间进度`(`--down`)。
- 月环用 `--primary`,年环用 `--accent-warm`——月蓝年橙是恒定语义。

### 3.2 横向目标进度条 + 时间进度虚线(最强签名)
- 每行:名称(左)→ 渐变进度条(当前值+达成率标在条上/条尾)→ 右侧两行小字 `目标:x` `差额:y`。
- 关键:一条贯穿所有行的**红色竖直虚线**标记「时间进度」位置,并在顶部标注「时间进度:51%」。条头是否越过虚线,一眼判断超前/滞后。
- 条渐变:左浅右深;月度蓝、年度橙。

### 3.3 进度药丸(顶栏)
- 页头下方或角落一枚胶囊:红色小标签「月业绩进度(结算口径)」+ 内嵌水平进度条 + 当前值/目标值。
- 何时用:日报,让当月大盘进度常驻可见。

### 3.4 双指标对置条(蝶形/发散)
- 共享中轴分类,左右各一指标:同比%(细条) vs 绝对值(粗条);或收入(主题色,向右) vs 支出(`--down`,向左)。
- 何时用:对比强关联双指标;收支、进出、正负。

## 4. 表格

- 表头:`--tint` 底色 + `--text-2` 加粗 12px;可带排序图标。
- 行:斑马纹(白/`#FAFBFC`),行高 36–40px;悬浮高亮 `--tint` 半透明。
- 合计行:置顶或置底,加 `--tint` 底色加粗。
- 数值列右对齐,tabular-nums;涨跌值直接着 `--up`/`--down` 色,可带 ↑↓。
- 列组分带:多指标家族时,给列组表头加不同浅底色(如「早降后1年内」黄带、「2年内」青带)区分口径组。
- 嵌入迷你条:占比/进度列可内嵌 60px 宽渐变微条。
- 明细表原则:一张看板最多一两张大明细表,放章节末尾兜底。

## 5. 筛选与交互控件

- 全局筛选:页头右侧,日期选择器 + 下拉,白底 `--hairline` 描边圆角 4px。
- 快捷时间 chips:「昨日/本周/本月/本年/最近7天/最近30天」一行药丸,选中态 `--primary` 底白字。
- Tab 切换:「活动分析 | 活动对比 | 活动明细」,选中 `--primary` 底白字药丸,未选中透明底 `--text-2`。
- 联动提示(签名细节):在可联动图表下方放一行 12px 红字「【下方数据点击即可相互联动】」;联动维度行放一排浅底 chips。
- 日历面板:单列侧栏放日历控件,选中日 `--primary` 方块,用于按日巡检场景。

## 6. 文本注释组件(可选,克制使用)

- 指标定义侧栏/脚注:11–12px `--text-3`,「指标定义:履约率=履约数/预约单数」;放卡片右上角 ⓘ 或页面顶部说明条。
- 洞察短句块:浅 `--tint` 底圆角块,每行「**彩色加粗前缀:**正文」(如「利润提升: 利润率从19.8%提升至23.5%」)。仅在汇报型看板使用,监控型看板让数字自己说话。

## 7. 装饰规则

装饰只允许出现在三处,其余一律留白:
1. 页头:六边形网格纹理、淡色光斑、LOGO。
2. KPI 卡底:超淡水印图标(不透明度 ≤6%)。
3. 分区丝带的柔和投影。

禁止:大面积插画、3D 效果、厚投影、彩色卡片底(除 `--tint` 与 wash 卡)、每数字加图标。密度已经很高,克制即高级。

# 版式规范 — Wyexin BI 页面解剖与排版逻辑

本文件回答两个问题:**一行到底放几个、每个多宽多高**(行型库);以及**怎么保证不错乱不溢出**(装配质检)。这是本 skill 的核心,布置页面前必读。

## 目录
1. 网格基础
2. 行型库(一行放几个,查表即用)
3. 宽度算法(类目数决定图表最小宽度)
4. 高度档位
5. 四种页面范式
6. 图表溢出与错乱质检(强制)

---

## 1. 网格基础

- 画布 1920(内容区 1888),12 列心智网格,卡距 12px,页边距 16px。
- 卡片宽度只允许:全宽、1/2、1/3、2/3、1/4、1/6(避免 1/5 等奇数比例)。
- **同行必等高**;行内混放不同组件时,以图表卡高度为准拉平。
- 一屏(1080 高)≤9 张卡;超过就用分区丝带开新章。

## 2. 行型库

按内容类型直接套用,不要自由发挥:

| 行型 | 组成 | 每卡宽 | 行高 | 用途 |
|---|---|---|---|---|
| KPI-4 | 4 张涨跌卡 | 1/4 | 110–130px | 默认 KPI 行,4–5 个核心指标 |
| KPI-6 | 6 张裸数字/涨跌卡 | 1/6 | 100–110px | 指标多而单一(日报);数字≤7位 |
| KPI-chips | 1 张卡内 6–9 枚强度 chip | 全宽 | 90px | 日报顶栏指标链 |
| KPI-组 | 3 张日/月/年三联组卡 | 1/3 | 150–170px | 驾驶舱首屏多维度 |
| 目标-环 | 月环面板 + 年环面板 | 1/2+1/2 | 220–240px | 目标达成区,月蓝年橙成对 |
| 趋势-全宽 | 1 张签名组合图 | 全宽 | 380–440px | 月趋势+目标+达成率(类目≥8) |
| 趋势-双 | 2 张折线/面积图 | 1/2+1/2 | 300–320px | 双主题并行趋势(如业绩/利润) |
| 进度-双 | 月进度条组 + 年进度条组 | 1/2+1/2 | 随行数,每行28px+ | 分线/分部门达成 |
| 构成-3 | 环图/面积/排名任取 3 | 1/3×3 | 280–300px | 构成与排名章,标题句式统一 |
| 构成-2+1 | 大图 + 窄图 | 2/3+1/3 | 300–340px | 桑基/热力表 配 环图/排名 |
| 图表+表 | 图 + 关联小表 | 2/3+1/3 或上图下表同卡 | 320–400px | 图表钻取明细 |
| 表-全宽 | 1 张明细表 | 全宽 | 自适应 | 章节末尾兜底,每板≤2张 |
| 小多图 | 行标签+迷你趋势 ×N 行 | 1/2 或 2/3 | N×60px | 多对象同指标对比 |

组行成章的默认节拍(驾驶舱):`KPI 行 → 目标-环 → 趋势-全宽 → 进度-双 → [丝带] → 构成-3 → 表-全宽`。日报节拍:`KPI-chips → 进度+近N日趋势(1/2+1/2) → 构成-3 → 排名-3`。

## 3. 宽度算法(类目数决定图表能放多窄)

放图前先数类目,再定宽度——这是防错乱的第一道闸:

| 图型 | 最小空间需求 | 结论 |
|---|---|---|
| 签名组合图(3柱+2线) | 每类目 ≥90px | 12 个月 → 只能全宽;≤6 类目才可 1/2 |
| 普通分组柱(2柱) | 每类目 ≥60px | 12 类目 → ≥1/2;≤6 类目可 1/3 |
| 单系列柱 | 每类目 ≥40px | 12 类目可 1/2,30 天日趋势 → 全宽或改折线 |
| 折线/面积 | 每点 ≥18px | 30 点 → ≥1/2;90 点 → 全宽且隔项标签 |
| 横向排名条 | 每行 ≥28px 高 | TOP15 → 高≥420px;1/3 宽卡建议 TOP≤8 |
| 环图 | 直径 ≥170px;引出标签再 +90px/侧 | 1/3 卡(约600px)可容引出标签;<300px 宽禁用引出改图例 |
| 漏斗/雷达 | ≥1/3 宽 | 维度 5–6 |
| 桑基/热力表 | ≥1/2 宽 | 节点/列多时 2/3 |
| 散点象限 | ≥1/2 宽 | 需要留象限标注位 |
| 地图 | ≥1/3 宽,高≥300px | 只标 TOP 省份 |

类目超限的处理顺序:①截断为 TOP-N ②隔项显示标签 ③换更宽的行型 ④拆成两张图。**禁止**斜排标签、禁止缩小字号到 10px 以下硬塞。

## 4. 高度档位

图表容器必须**固定高度**(防止 auto 高度导致的塌陷/抖动):

- KPI 卡 110 / 130
- 环面板 220
- 小图 260(1/3 宽标配)
- 中图 320(1/2 宽标配)
- 大图 420(全宽组合图标配)
- 表格行高 36–40,表头 40

## 5. 四种页面范式

| 看板性质 | 范式 | 章节骨架 |
|---|---|---|
| 高管大盘 | A 驾驶舱 | KPI→目标环→趋势→进度条→[丝带]→构成→明细表 |
| 每日速览 | B 日报 | 进度药丸+KPI chips→目标条+近N日趋势→环图×3→排名×3,1–1.5 屏内结束 |
| 专项巡检 | C 监测 | 筛选(侧栏180px或顶栏)→日/月/年三联→主监控图→异常双表→前后14日对照→大明细表 |
| 多分支同构 | D 分中心 | [蓝渐变横幅+同构四联面板]×N 纵向复制,面板内容顺序高度完全一致 |

通用规则:左总右分、上总下分;月/年成对镜像;卡片标题=业务问题名词化;移动端单列、KPI 2×N 宫格、砍明细表。

## 6. 图表溢出与错乱质检(交付前强制执行)

### 6.1 ECharts 配置层防线(写代码时就要做对)

1. `grid.containLabel:true` 一律开启。
2. **顶部留头**:数据标签在柱顶时,`yAxis.max: v=>v.max*1.15`(单层)~`1.25`(标签较长);组合图双率线悬浮时 `grid.top≥110` 且第二 y 轴 min/max 校准,率线不得压柱、不得出顶。
3. **右侧留尾**:横向条值标签在条尾外侧时 `grid.right≥48`;数值 ≥6 位再加。
4. **类目名截断**:y 轴类目名 >6–8 字用 formatter 截断加省略号,tooltip 里给全名;x 轴类目过密用 `interval:'auto'`,禁止 rotate。
5. **环图标签**:两行式引出标签仅在卡宽 ≥300px 时用;否则 `label.show:false` + 图例;`labelLine` 长度 12/8;防重叠开 `avoidLabelOverlap:true`。
6. **数字缩写**:标签值 ≥6 位数换算为「万/亿」保留 1 位小数;KPI 大数字 >7 位时降档字号(34→28→24)或换单位,禁止溢出卡片。
7. **固定容器高**:每个 chart div 都有确定 height;`window.resize → chart.resize()` 必须挂;多图页面初始化在 DOM ready 之后。
8. 图例过多(>6 系列)时:合并系列或分拆图表,不许让图例挤占绘图区超过两行。

### 6.2 装配层检查(HTML 拼好后过一遍)

- [ ] 同行卡片顶底对齐,无参差(逐行扫)
- [ ] 1920 与 1440 两档宽度各看一遍:无横向滚动条、无标签互相压盖、无柱形溢出卡片
- [ ] 每张图表:数据标签完整可读(首、尾、最大值三处重点看)
- [ ] 横向排名:最长类目名 + 最大数值同时在卡内
- [ ] 环图:引出标签没有交叉、没有伸出卡片
- [ ] 组合图:达成率线、同比线分层清晰,没有和柱/图例打架
- [ ] 表格:长文本省略号生效,数值列右对齐,表头不换行
- [ ] KPI:最大数字 + 涨跌幅在卡内一行放得下
- [ ] 空数据/零值:图表不报错、不显示 NaN
- [ ] 字号自检:卡片标题 16px、轴与数据标签 11px、正文 12px,无 <10px 文字

### 6.3 检查方法

- **第一优先**:HTML 产物内置 QA——页面 URL 加 `?qa=1`,自动检测横向溢出/卡片参差/图表文本越界/异常字号,标红并出报告。
- 有浏览器/截图能力时:打开页面截图,按 6.2 清单逐项目检,发现问题回到 6.1 修配置。
- 无渲染能力时:用 Node 模拟执行所有 option 构建函数,校验无异常、无 NaN;再逐条静态核对 6.1 的 8 条防线是否在代码中落实。
- 修改数据后必须重查:溢出多数由"换了一组更长的数据"引起。
